## 平台与账号对应关系

SendFox 涉及两类平台账号：

| 用途 | 平台 | 获取方式 |
|------|------|----------|
| 内容采集 | 内容社区 | 桌面客户端抓包（微信小程序） |
| 内容发布 | 目标社区 | 手机验证码登录 |

## 桌面客户端抓包

客户端通过 HTTPS 中间人代理拦截**微信小程序**的网络请求，自动提取认证 Token。

### 系统要求

- **macOS**：macOS 13 (Ventura) 及以上
- **Windows**：推荐 Windows 11

### 安装证书

客户端会在首次使用时自动生成 CA 证书（`NodeMITMProxyCA`），用户只需授权安装：

- **macOS**：点击「生成并安装证书」，弹出系统密码框，输入密码即可完成。大部分情况下自动完成信任设置
- **Windows**：点击「生成并安装证书」，弹出 UAC 确认框，点击「是」即可
- **macOS Ventura+**：如自动信任失败，需手动在「钥匙串访问」中双击证书，将信任设为「始终信任」

### 抓包步骤

1. 确保证书已安装并信任
2. 在客户端选择平台（内容社区 / 目标社区）
3. 点击「开始抓包」
4. 在手机微信中打开对应平台的小程序，浏览内容
5. 客户端自动识别并提取 Token
6. 在「已识别 Token」列表中确认
7. 点击「上传」同步到管理后台

> 抓包仅适用于微信小程序，不支持独立 App。抓包完成后请点击「停止抓包」恢复正常网络。

## 目标社区手机验证码登录

目标社区平台提供免抓包的快速绑定方式：

1. 进入管理后台「账号管理」页面
2. 点击「账号登录」
3. 输入手机号，获取验证码
4. 验证成功后账号自动创建

## 账号状态

| 状态 | 说明 | 处理方式 |
|------|------|----------|
| ACTIVE | 账号正常可用 | 无需操作 |
| EXPIRED | Token 已过期 | 重新使用验证码登录 |
| INVALID | 凭证无效 | 删除后重新绑定 |

## 马甲号说明

马甲号（Bump User）是目标社区平台的功能，**仅限圈子管理员账号使用**：

- 如果你的目标社区账号是某个圈子的管理员，系统会自动同步该圈子的马甲号列表
- 马甲号可在发布任务中使用，实现每轮发布自动切换身份
- 非管理员账号无法使用马甲号功能

## 常见问题

### macOS 证书安装后仍然无法抓包？

确认已在「钥匙串访问」中将证书信任级别设为「始终信任」，然后重启客户端。

### 抓包没有捕获到 Token？

- 确认是在微信小程序中操作（不是独立 App）
- 确认证书已正确安装和信任
- 确认客户端显示「抓包中」状态
- 尝试在小程序中退出登录后重新登录

### Token 上传后状态为 INVALID？

可能是抓包后等待时间过长导致 Token 过期，建议抓包后立即上传。
