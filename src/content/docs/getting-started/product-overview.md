## SendFox 是什么

SendFox 是一个面向高校社区运营的自动化工具。它帮你从内容社区自动采集优质帖子，经过 AI 审核和改写后，分发到你要运营的目标社区。

简单来说：**从 A 社区搬内容，改头换面后发到 B 社区，全程自动化。**

## 核心工作流程

```
采集内容          AI 处理            发布分发
┌─────────┐    ┌──────────┐    ┌──────────┐
│ 内容社区 │───→│ 审核+改写 │───→│ 目标社区 │
│ (源)     │    │          │    │ (目标)   │
└─────────┘    └──────────┘    └──────────┘
  自动采集       敏感词过滤       定时发布
  图片下载       AI 合规判断      马甲号轮换
  去重存档       AI 改写去重      多学校并发
```

1. **采集**：使用你绑定的账号，按设定间隔自动从内容社区抓取新帖子，下载图片并去重存储
2. **审核**：先过敏感词库，再由 AI 判断内容是否合规，不合规的自动屏蔽
3. **改写**：发布前由 AI 对内容改写，避免不同学校出现完全相同的帖子
4. **发布**：按你的设定自动发布到目标社区，支持马甲号轮换和时段控制

## 关键概念

在开始使用之前，你需要理解以下几个核心概念：

### 两类平台

| 平台类型 | 作用 | 账号获取方式 |
|---------|------|-------------|
| **内容社区** | 内容的来源，从这里采集帖子 | 桌面客户端抓包（微信小程序） |
| **目标社区** | 内容的目的地，往这里发布帖子 | 手机验证码登录 |

> 可以理解为：内容社区是"货源"，目标社区是"销路"。

### 学校（School）

这里的"学校"指的是高校社区中的一个子社区（比如某大学的社区板块）。SendFox 围绕学校来组织内容：

- **采集学校**：你要从哪些学校采集内容（不受数量限制）
- **发布学校**：你要往哪些学校发布内容（需要占用名额）
- 一个采集学校的内容可以发布到多个发布学校

### 学校名额（Slot）

学校名额决定你能绑定多少个**发布学校**。名额来源于订阅套餐和额外购买的学校包，解绑学校后名额可以复用。

### 马甲号（Bump User）

马甲号是目标社区平台的功能，仅限**圈子管理员账号**使用。开启后，每轮发布会自动切换发布身份，避免被识别为同一操作者。

### Token

Token 是平台账号的身份凭证，采集和发布都需要有效的 Token。内容社区通过桌面客户端抓包获取，目标社区通过手机验证码登录获取。Token 会过期，过期后需要重新获取。

## 使用前提

开始之前，你需要准备：

- 一台电脑（macOS 13+ 或 Windows 11 推荐），用于运行桌面客户端抓包
- 一部手机（用于在微信小程序中操作）
- 一个 AI 服务商的 API Key（用于内容审核和改写）
- 内容社区和目标社区的账号

准备好这些后，进入 [快速上手](/docs/getting-started/quick-start/) 开始配置。
