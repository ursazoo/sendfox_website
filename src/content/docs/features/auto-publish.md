## 工作原理

发布任务从已审核通过的帖子池中选取内容，自动发布到配置的目标社区：

1. 按照发布目标的优先级选择下一篇帖子
2. 优先匹配与发布学校相关的帖子
3. 如开启 AI 改写，先对内容进行改写
4. 上传图片（如平台需要）
5. 使用对应平台的签名算法发布内容
6. 记录发布结果

## 创建发布任务

1. 进入「发布任务」页面
2. 点击「创建任务」
3. 添加发布目标
4. 配置发布参数
5. 保存并启动

## 发布目标

每个发布目标由以下要素组成：

- **目标学校**：内容发布到哪个学校社区
- **发布账号**：使用哪个账号进行发布
- **马甲号**（目标社区）：使用哪个马甲号身份发布

一个发布任务可以同时发布到多个学校。

## 马甲号轮换

马甲号（Bump User）是目标社区平台的功能，**仅限圈子管理员账号使用**：

- 仅当你绑定的目标社区账号是某个圈子的管理员时，才能使用马甲号
- 马甲号从平台 API 自动同步
- 每轮发布会自动切换到下一个马甲号
- 如马甲号失效，系统会自动跳过
- 非管理员账号不会显示马甲号选项

## 发布参数

| 参数 | 说明 | 建议 |
|------|------|------|
| 发布间隔 | 两篇帖子之间的时间间隔 | 5-15 分钟 |
| 轮次间隔 | 一轮发布完毕后的休息时间 | 1-2 小时 |
| 发布时段 | 允许发布的时间范围 | 避开深夜 0-6 点 |

## AI 改写

开启 AI 改写后，系统会在发布前对内容进行处理：

- 自动改写标题和正文
- 保留核心信息，改变表达方式
- 避免不同学校发布完全相同的内容
- 改写后的内容会保存记录，便于回溯

## 发布状态

| 状态 | 说明 |
|------|------|
| PENDING | 等待发布 |
| PUBLISHING | 发布中 |
| PUBLISHED | 发布成功 |
| FAILED | 发布失败（可重试） |

## 常见问题

### 发布失败怎么办？

- 检查账号 Token 是否过期
- 检查马甲号是否有效
- 查看失败原因，针对性处理
- 可以在记录列表中点击「重试」

### 如何避免发布太频繁被平台检测？

- 设置合理的发布间隔（不低于 5 分钟）
- 开启马甲号轮换
- 开启 AI 改写，避免内容重复
- 设置发布时段，避开低活跃时间
