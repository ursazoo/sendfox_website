## 学校是怎么产生的

学校记录是系统自动生成的，**不需要手动添加**。当你在桌面客户端抓包并上传账号时，选择对应的学校后，系统会自动创建学校记录。

每个学校记录包含以下信息：

- 所属平台
- 学校名称（关联全国高校主数据）
- 配置类型：仅采集 / 仅发布 / 采集 + 发布

> 采集学校不受名额限制，可以自由上传。发布学校需要占用学校名额（Slot）。

## 订阅套餐

### 名额机制

学校名额（Slot）是订阅的核心计量单位：

- 每个名额允许绑定一所发布学校
- 名额数量由订阅套餐决定
- 名额可复用：解绑学校后名额释放

### 购买订阅

1. 进入「订阅」页面
2. 选择合适的套餐
3. 微信扫码支付
4. 支付成功后名额立即到账

### 学校包

如果套餐赠送的名额不够，可以额外购买学校包：

- 在「订阅」页面点击「购买学校包」
- 学校包跟随套餐有效期

## 名额绑定流程

1. 购买订阅获得学校名额
2. 进入「学校名额」页面，查看可用名额
3. 将名额绑定到想要发布的学校
4. 绑定完成后，上传该学校的发布账号时系统自动创建学校记录

> 解绑学校：在名额列表中点击「解绑」，名额释放后可重新分配给其他学校。

## 学校管理

学校管理页面用于查看和管理已有的学校配置：

- 查看所有学校列表，支持按平台和配置类型筛选
- 编辑学校配置信息
- 管理采集学校和发布学校之间的绑定关系

### 采集-发布绑定

你可以将采集学校和发布学校建立绑定关系：

- 采集到的内容自动分配给绑定的发布学校
- 支持一对多绑定（一个采集学校对应多个发布学校）
- 发布时优先选取与目标学校相同的内容，无同校内容时自动回落到其他待发布帖子
